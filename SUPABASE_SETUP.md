
# ⚡ Supabase Setup Instructions (IMPORTANT)

To make the registration form work, you need to set up your Supabase project.

### 1. Create Supabase Project
1. Go to [supabase.com](https://supabase.com) and sign in.
2. Click "New Project".
3. Name it `30-hour-sprint` and give it a database password.
4. Select a region close to you (e.g., Mumbai, Singapore).

### 2. Create the Database Table
1. Go to the **SQL Editor** (icon with `>_` on the left sidebar).
2. Click "New Query".
3. Paste the following SQL code and click **Run**:

```sql
create table registrations (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  full_name text not null,
  email text not null,
  mobile text not null,
  age integer,
  gender text,
  city text,
  state text,
  country text,
  education_status text,
  organization text,
  experience_level text,
  github text,
  linkedin text,
  portfolio text,
  skills text[] default '{}',
  participation_type text,
  team_name text,
  team_size integer,
  team_leader_name text,
  team_leader_email text,
  team_leader_phone text,
  has_idea boolean default false,
  project_title text,
  problem_statement text,
  idea_desc text,
  tech_stack text[] default '{}',
  full_commitment boolean default true,
  voice_sessions text,
  motivation text,
  expectations text
);
```

### 3. Disable Row Level Security (For Quick Testing)
For this hackathon, since we want anyone to be able to register without logging in first:
1. Go to **Table Editor** -> `registrations`.
2. Click "RLS is enabled" (top right) -> **Disable RLS**.
   *(Note: In a production app with auth, you would enable RLS and add policies. For a public form, disabling it or adding a "public insert" policy is fine).*

### 4. Get API Keys
1. Go to **Project Settings** (gear icon) -> **API**.
2. Copy the `Project URL`.
3. Copy the `anon` / `public` key.

### 5. Add to Project
1. Create a file named `.env` in your project root (`d:\30-hour-learn-&-build-sprint\.env`).
2. Paste the keys like this:

```env
VITE_SUPABASE_URL=your_project_url_here
VITE_SUPABASE_ANON_KEY=your_anon_key_here
```

3. Restart your dev server (`Ctrl+C` then `npm run dev`).

✅ **Done! Your registration form will now save data to the dashboard.**
